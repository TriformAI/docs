## What are API Keys?

API Keys are secret tokens that authenticate requests to your deployed Projects. They verify that the caller has permission to execute your Project and track usage for billing and monitoring.

## Creating API Keys

### Via UI

1. Go to **Profile → API Keys** (or Organization → API Keys)
2. Click **Create New Key**
3. Configure:
   - **Name:** `Production Web App`
   - **Description:** Key for main web application
   - **Permissions:** Execute Projects
   - **Scope:** Specific Projects or all
   - **Rate limit:** Custom limits (optional)
   - **Expiration:** Never, or specific date
4. Click **Create**
5. **Copy the key immediately** — It's only shown once!

**Generated key example:**
```
tri_sk_live_abc123def456ghi789jkl012
```

### Via API

```bash
POST /v1/api-keys

{
  "name": "Production Web App",
  "permissions": ["projects:execute"],
  "scope": "all",
  "rate_limit": 1000,
  "expires_at": null
}
```

**Response:**
```json
{
  "key": "tri_sk_live_abc123def456ghi789jkl012",
  "id": "key_xyz789",
  "created_at": "2025-10-01T10:30:00Z"
}
```

## Key types

### User API Keys

Tied to your personal account.

**Characteristics:**
- Full access to your Projects
- Inherits your permissions
- Can't be shared safely
- Used for personal tools, scripts

**Prefix:** `tri_sk_user_`

### Organization API Keys

Shared across team, scoped to Organization.

**Characteristics:**
- Team-wide access
- Fine-grained permissions
- Can be Project-specific
- Used for applications, integrations

**Prefix:** `tri_sk_org_`

### Service API Keys

For specific services or applications.

**Characteristics:**
- Limited scope (one Project)
- Restricted permissions
- Independently revocable
- Used for third-party integrations

**Prefix:** `tri_sk_svc_`

### Public API Keys

For client-side usage (read-only, limited).

**Characteristics:**
- Safe to expose publicly
- Read-only operations
- Rate-limited heavily
- No sensitive data access

**Prefix:** `tri_pk_` (publishable key)

## Using API Keys

### HTTP header

Most common method:

```bash
curl -X POST https://api.triform.ai/v1/your-project/execute \
  -H "Authorization: Bearer tri_sk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"input": "data"}'
```

### Query parameter (not recommended)

For environments where headers are difficult:

```bash
curl "https://api.triform.ai/v1/your-project/execute?api_key=tri_sk_live_abc123..."
```

**Warning:** Query parameters appear in logs. Use headers when possible.

### SDK usage

**Python:**
```python
from triform import Triform

client = Triform(api_key="tri_sk_live_abc123...")
result = client.projects.execute("your-project", {"input": "data"})
```

**JavaScript:**
```javascript
import { Triform } from 'triform-sdk';

const client = new Triform({ apiKey: 'tri_sk_live_abc123...' });
const result = await client.projects.execute('your-project', { input: 'data' });
```

## Permissions

Control what each API key can do.

### Permission types

| Permission | Description |
|------------|-------------|
| `projects:execute` | Execute Projects |
| `projects:read` | View Project details |
| `projects:write` | Edit Projects |
| `projects:deploy` | Deploy Projects |
| `projects:delete` | Delete Projects |
| `executions:read` | View execution history |
| `variables:read` | Read Global Variables (config only, not secrets) |
| `variables:write` | Create/update Global Variables |
| `api-keys:read` | List API keys |
| `api-keys:write` | Create/revoke API keys |

### Permission scopes

**All Projects:**
```json
{
  "permissions": ["projects:execute"],
  "scope": "all"
}
```

**Specific Projects:**
```json
{
  "permissions": ["projects:execute"],
  "scope": ["project_123", "project_456"]
}
```

**Read-only access:**
```json
{
  "permissions": ["projects:read", "executions:read"],
  "scope": "all"
}
```

### Principle of least privilege

Grant only necessary permissions:

**❌ Too broad:**
```json
{
  "permissions": ["*"],  // All permissions
  "scope": "all"         // All Projects
}
```

**✅ Appropriate:**
```json
{
  "permissions": ["projects:execute"],
  "scope": ["customer-support-bot"]
}
```

## Managing API Keys

### Viewing keys

1. Go to **Profile → API Keys**
2. See list of all your keys

**Each entry shows:**
- Name and description
- Key ID (not the secret)
- Permissions
- Created date
- Last used
- Status (active, revoked, expired)

**Note:** Full key value is never shown again after creation.

### Rotating keys

Periodically update keys for security.

**Process:**
1. Create new key with same permissions
2. Update applications to use new key
3. Test thoroughly
4. Revoke old key once confirmed

**Recommended frequency:** Every 90 days

### Revoking keys

Immediately disable a key:

1. Go to API Keys
2. Select key
3. Click **Revoke**
4. Confirm

**Effect:** Immediate. All requests with this key will fail.

**When to revoke:**
- Key compromised or leaked
- Employee left team
- Service no longer needed
- Migrated to new key

**Cannot be undone:** Revoked keys cannot be reactivated.

### Viewing key usage

See how keys are being used:

1. Select key
2. Click **Usage**
3. View:
   - Requests per day
   - Which Projects executed
   - Success/failure rate
   - Geographic distribution
   - Error types

**Use case:** Detect unusual activity, optimize usage.

## Rate limiting

Control request frequency per key.

### Default limits

**Free tier:**
- 60 requests per minute
- 1,000 requests per hour
- 10,000 requests per day

**Pro tier:**
- 600 requests per minute
- 10,000 requests per hour
- 100,000 requests per day

**Enterprise:** Custom limits

### Custom rate limits

Set per-key limits:

```json
{
  "rate_limit": {
    "per_minute": 100,
    "per_hour": 5000,
    "per_day": 50000
  }
}
```

**Use case:** Different limits for different services.

### Handling rate limits

**Response when exceeded:**
```json
{
  "error": "rate_limit_exceeded",
  "message": "Rate limit exceeded. Try again in 30 seconds.",
  "retry_after": 30,
  "limit": 100,
  "remaining": 0,
  "reset_at": "2025-10-01T10:31:00Z"
}
```

**Status code:** 429 Too Many Requests

**Headers:**
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1696158060
Retry-After: 30
```

**Client handling:**
```python
import time
import requests

response = requests.post(url, headers=headers, json=payload)

if response.status_code == 429:
    retry_after = int(response.headers.get('Retry-After', 60))
    print(f"Rate limited. Waiting {retry_after} seconds...")
    time.sleep(retry_after)
    # Retry request
```

## Expiration

Set API keys to auto-expire.

### Why use expiration?

- **Security:** Limit lifetime of leaked keys
- **Compliance:** Meet audit requirements
- **Temporary access:** Short-term integrations

### Setting expiration

**Fixed date:**
```json
{
  "expires_at": "2025-12-31T23:59:59Z"
}
```

**Relative:**
```json
{
  "expires_in_days": 90
}
```

**Never expires:**
```json
{
  "expires_at": null
}
```

### Expiration notifications

Get notified before keys expire:

- 30 days before
- 7 days before
- 1 day before
- On expiration

**Notification channels:** Email, Slack, webhook

## Security best practices

> **Never commit keys to git** — Use environment variables

> **Rotate keys regularly** — Every 90 days

> **Revoke unused keys** — Remove keys no longer needed

> **Use minimal permissions** — Grant only what's necessary

> **Monitor key usage** — Watch for suspicious activity

> **Set expiration dates** — Limit key lifetime

> **Store securely** — Use secret managers (AWS Secrets, Vault)

> **Don't share keys** — Each service gets its own key

## Key storage

### Environment variables

**Recommended method:**

```bash
# .env file (don't commit!)
TRIFORM_API_KEY=tri_sk_live_abc123...
```

**Usage:**
```python
import os

api_key = os.environ.get('TRIFORM_API_KEY')
```

### Secret managers

**AWS Secrets Manager:**
```python
import boto3

client = boto3.client('secretsmanager')
response = client.get_secret_value(SecretId='triform-api-key')
api_key = response['SecretString']
```

**HashiCorp Vault:**
```python
import hvac

client = hvac.Client(url='https://vault.example.com')
secret = client.secrets.kv.read_secret_version(path='triform-api-key')
api_key = secret['data']['data']['key']
```

### What NOT to do

**❌ Hardcode in source:**
```python
api_key = "tri_sk_live_abc123..."  # BAD!
```

**❌ Commit to git:**
```
git add config.py  # Contains API key - BAD!
```

**❌ Share in chat/email:**
```
"Hey, use this key: tri_sk_live_abc123..."  # BAD!
```

**❌ Client-side code:**
```javascript
// Never expose secret keys in browser JavaScript
const apiKey = "tri_sk_live_abc123...";  // BAD!
```

## Troubleshooting

**Problem:** API key not working  
**Solution:** Check key is correct, not revoked, has required permissions, and is for the right environment

**Problem:** Rate limit hit too quickly  
**Solution:** Implement request batching, caching, or request higher limits

**Problem:** Can't find my API key  
**Solution:** Keys are only shown once. Create a new one if lost.

**Problem:** Key works locally but not in production  
**Solution:** Verify environment variables are set correctly in production
