## Versioning

Triform uses semantic versioning (MAJOR.MINOR.PATCH):

- **MAJOR** â€” Breaking changes or significant new capabilities
- **MINOR** â€” New features, backwards-compatible
- **PATCH** â€” Bug fixes and minor improvements

## Release schedule

- **Major releases:** Quarterly (every 3 months)
- **Minor releases:** Monthly
- **Patch releases:** As needed (typically weekly)

## Deprecation policy

When we deprecate features:

1. **Announce** â€” At least 90 days notice
2. **Document** â€” Migration guide provided
3. **Support** â€” Old feature continues working during deprecation period
4. **Remove** â€” Only after notice period and next major version

## Beta features

Features marked **(Beta)** are:
- Functional but may have bugs
- Subject to changes
- Available to test and provide feedback
- Not recommended for production-critical workloads
